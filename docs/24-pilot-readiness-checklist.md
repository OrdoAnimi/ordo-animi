# Pilot Readiness Checklist

Version: 0.1
Status: Private checklist

## Purpose

This checklist defines what must be ready before VALOUR is shown to early users, advisors, or potential funders.

## Product readiness

- Product charter exists.
- Founder story exists.
- System architecture exists.
- Market positioning exists.
- MVP plan exists.
- Scenario library exists.
- Prompt pack exists.
- Pilot templates exist.
- Pilot pack exists.

## Message readiness

- One-line definition is clear.
- Short explanation is clear.
- Long explanation is clear.
- Landing page draft exists.
- Outreach copy exists.
- Discovery call notes exist.
- Funding one-pager exists.

## Pilot readiness

- Pilot intake form exists.
- Scenario selection method exists.
- Rehearsal process exists.
- After-action review process exists.
- Pattern snapshot template exists.
- Completion report template exists.
- Validation tracker exists.

## Evidence readiness

- Feedback capture method exists.
- Testimonial capture method exists.
- Validation criteria exist.
- Decision gate exists.
- Risk and IP notes exist.

## First contact readiness

Before contacting early users, prepare:

- Target contact list.
- One short outreach note.
- Pilot pack link or extract.
- Discovery call script.
- Follow-up template.
- Validation tracker row for each contact.

## Do not proceed if

- The product is being explained as the full Ordo Animi platform instead of VALOUR.
- The offer sounds like generic leadership training.
- The user is unclear what they need to bring.
- There is no real upcoming scenario to test.
- Feedback will not be captured.

## Ready state

Pilot Readiness is complete when the first five target contacts can be approached without needing new product explanation work.
