# VALOUR Capability Map

Version: 0.1
Status: Private capability map
Phase: Architecture and Pilot Build

## Purpose

This document defines the capability map for VALOUR for Architecture Leadership.

The capability map bridges enterprise architecture, business architecture, solution architecture, product backlog, and implementation planning.

## Capability layers

VALOUR capabilities are grouped into seven layers:

1. Market and customer capability
2. Pilot operating capability
3. Leadership scenario capability
4. Agentic AI capability
5. Data and evidence capability
6. Security and governance capability
7. Platform and delivery capability

## 1. Market and customer capability

### Customer targeting

Identify architecture leaders, technical leaders, pilot users, advisors, and potential team buyers.

### Offer management

Define pilot offers, individual pilot options, team pilot options, and later commercial packages.

### Outreach and engagement

Manage messaging, discovery conversations, follow-ups, and pilot invitations.

### Feedback and conversion

Capture feedback, pilot interest, next actions, and commercial signals.

## 2. Pilot operating capability

### Intake management

Capture user role, context, leadership challenge, scenario, and baseline confidence.

### Scenario selection

Map user situation to the right VALOUR scenario.

### Pilot workflow management

Move the user through intake, preparation, rehearsal, review, and pattern reporting.

### Operator workspace

Provide the founder or operator with the working materials required to run the pilot manually or semi-automatically.

### Pilot reporting

Produce user-facing summaries and internal pilot evidence.

## 3. Leadership scenario capability

### Scenario catalogue

Maintain architecture leadership and adjacent leadership scenarios.

### Scenario templates

Define preparation, rehearsal, language refinement, and review structures for each scenario.

### Scenario variants

Adapt standard scenarios to specific user contexts without losing the core workflow.

### Scenario governance

Keep scenarios focused, useful, and aligned to the first commercial wedge.

## 4. Agentic AI capability

### Intake guide

Helps structure the user situation.

### Scenario selector

Recommends the best scenario based on user context.

### Preparation guide

Creates a useful preparation brief.

### Rehearsal guide

Simulates the leadership moment and gives feedback.

### Language refinement guide

Improves user responses for clarity, cadence, empathy, authority, and decision framing.

### Review guide

Runs the after-action review.

### Pattern reporter

Identifies recurring leadership patterns and next development focus.

### VALLUM handover guide

Supports professional closure and later handover into VITA.

## 5. Data and evidence capability

### User records

Maintain pilot user information with minimum data capture.

### Scenario records

Store selected scenario, context, and preparation details.

### Rehearsal records

Capture prompt outputs, user answers, and improvements.

### Review records

Capture after-action review and real-world feedback.

### Pattern records

Capture recurring patterns and recommendations.

### Validation metrics

Track completion, usefulness, confidence shift, repeat intent, and next action.

## 6. Security and governance capability

### Privacy control

Limit unnecessary collection of confidential information.

### Access control

Keep private product and pilot material restricted.

### IP control

Protect source prompts, architecture, founder material, and pilot evidence.

### Data retention

Define what is stored, where it is stored, and when it should be removed.

### Architecture decision governance

Record major architecture decisions in an architecture decision log.

## 7. Platform and delivery capability

### Source of truth

Use GitHub as the private product and architecture source of truth.

### Documentation system

Maintain product, architecture, pilot, and release documents.

### Pilot interface

Provide a manual, form-based, or lightweight web interface for the pilot.

### AI integration

Provide access to selected AI models or model gateway.

### Workflow orchestration

Move from manual prompt operation to structured agent workflow.

### Deployment pipeline

Enable future build, test, release, and deployment.

## Capability maturity

### Current maturity

Documentation and manual operating model exist.

### Next maturity

Pilot-ready manual workflow with scenario prompts, runbook, evidence capture, and architecture backlog.

### Future maturity

Lightweight product interface, semi-automated prompt flow, agentic orchestration, user records, and reporting.

## Capability priority

Highest priority:

- Pilot workflow management
- Scenario selection
- Preparation guide
- Rehearsal guide
- Review guide
- Evidence capture

Medium priority:

- Landing page
- Intake form
- Pattern report generation
- Validation dashboard

Later priority:

- Calendar integration
- Enterprise SSO
- Full multi-agent orchestration
- VITA productisation
- Desktop companion

## Capability conclusion

VALOUR does not need the full platform to begin.

It needs the core capability chain: identify user, choose scenario, prepare, rehearse, refine, review, report, and capture evidence.
