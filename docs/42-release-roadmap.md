# VALOUR Release Roadmap

Version: 0.1
Status: Private release roadmap
Phase: Architecture and Pilot Build

## Purpose

This document defines the staged release roadmap for VALOUR for Architecture Leadership.

The roadmap moves from manual pilot to repeatable pilot, then to lightweight product, guided AI workflow, and team pilot readiness.

## Roadmap principle

Build the smallest release that proves the next risk.

Do not build platform depth before proving workflow value.

Do not build enterprise features before proving team demand.

Do not expand into the full Ordo Animi ecosystem before VALOUR proves the architecture leadership wedge.

## Release 0: Foundation Release

Status: complete.

Purpose: create the source-of-truth product workspace.

Outputs include product charter, founder story, system architecture, market positioning, GTM strategy, funder pack, MVP plan, scenario library, pilot templates, outreach material, and risk notes.

Success criteria: the product concept is contained and no longer moving around uncontrolled.

## Release 1: Pilot Ready Release

Status: in progress.

Purpose: create everything needed to run the first VALOUR pilot manually.

Outputs include pilot workflow, user journey, data model, manual runbook, evidence model, success metrics, architecture input brief, capability map, value stream map, and MVP/MLP definition.

Success criteria: one user can complete the full prepare, rehearse, review, and report loop manually.

## Release 2: Repeatable Pilot Release

Purpose: make the manual pilot repeatable across several users without reinventing the process.

Outputs include standard pilot folder structure, user record templates, scenario prompt packs, pattern report template, pilot completion report, validation review cadence, and contact tracker.

Success criteria: the pilot can be repeated with at least five users using the same operating model.

## Release 3: Lightweight Interface Release

Purpose: introduce a simple user-facing or operator-facing interface.

Outputs include landing page, intake form, scenario selection interface, prompt workflow page, report output, and evidence capture form.

Success criteria: the user or operator can complete the pilot with less manual assembly.

## Release 4: Guided AI Workflow MVP

Purpose: convert manual prompt flow into a structured guided workflow.

Outputs include intake guide, scenario selector, preparation guide, rehearsal guide, language refinement guide, review guide, pattern reporter, and basic workflow orchestration.

Success criteria: the product can guide the user through the core flow with less founder intervention.

## Release 5: VALOUR MLP

Purpose: make the experience polished, repeatable, and credible enough for early paid pilots or team use.

Outputs include improved interface, better report design, stronger scenario guidance, saved user patterns, team pilot summary, basic analytics, and clear pilot packaging.

Success criteria: users want to repeat the experience, recommend it, or use it with a team.

## Release 6: VALLUM Handover Release

Purpose: introduce the first mode-switching layer between professional leadership and personal recovery.

Outputs include end-of-day closure workflow, work residue capture, personal handover prompt, VITA entry point, boundary prompts, and recovery prompts.

Success criteria: users can close professional mode and transition into personal mode with less carryover.

## Release 7: Team Pilot Release

Purpose: prepare VALOUR for team pilot use.

Outputs include team onboarding, admin summary, privacy controls, team scenario reporting, capability lead pack, architecture pack, and security questionnaire baseline.

Success criteria: a small team or practice can run VALOUR as a structured pilot.

## Release 8: Platform Architecture Release

Purpose: select and document the full technical architecture for the product platform.

Outputs include architecture decision records, cloud platform decision, integration design, data architecture, security architecture, deployment architecture, and operating model.

Success criteria: the product has a clear build path for scale.

## Roadmap conclusion

VALOUR should move in controlled releases from manual proof to repeatable pilot, lightweight interface, guided workflow, and team pilot readiness.

The long-term Ordo Animi vision remains intact, but VALOUR must prove the first wedge before the broader system expands.
