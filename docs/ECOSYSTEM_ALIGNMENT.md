# SA Artefact Generator Ecosystem Alignment

## Role in the system

SA Artefact Generator is the solution architecture limb of the ZenCloud / Velocity ecosystem.

It owns delivery-level solution architecture outputs: solution overview, ADRs, interfaces, dependencies, trade-offs, solution risks, TRA readiness, and handover material.

## Alignment target

The maturity reference is EA Artefact Generator.

SA must reach the same baseline:

- Clear product intent.
- Live or deployable route.
- VAF ecosystem fit.
- Recognised professional artefact catalogue.
- Local deterministic generation.
- Future AI generation path.
- Artefact output management.
- Markdown copy and download.
- Structured export path.
- Artefact lifecycle states.
- Security and data notes.
- Local run, build, preview, and deploy instructions.
- Roadmap.
- Licence posture.

## Current capability

The current app already includes:

- SA Meeting Primer.
- Artefact Generator.
- SA Advisor.
- Artefact selection by solution phase.
- Field-level guidance.
- Local deterministic artefact generation.
- Optional `/api/generate` path when configured.
- Markdown copy and download.
- CSV/object export utilities.
- Situation-led entry points.

## Required uplift

To match EA maturity, SA needs:

- Cloudflare Pages deployment route.
- Custom domain target: `sa.velocityarchitecture.com.au`.
- Deployment documentation.
- Formal artefact lifecycle controls.
- Artefact library or output management view.
- PMO Portal handoff record.
- Security and data-flow documentation.
- Screenshots after live deployment.

## Handoff model

SA receives context from BA and EA, produces delivery-level solution architecture outputs, and hands reviewed artefacts to PMO Portal for lifecycle governance, client visibility, delivery control, and handover.

## Shared lifecycle

Required -> Draft -> In Review -> Approved -> Shared -> Locked -> Handover

Minimum immediate states:

- Draft.
- Reviewed.
- Approved.
