# Ecosystem Maturity Baseline v0.1

## Purpose

This document defines the minimum maturity standard for the ZenCloud / Velocity tool ecosystem.

The baseline reference is the EA Artefact Generator. Every specialist limb should reach the same practical level of clarity, deployment readiness, artefact structure, and ecosystem fit.

## System shape

- Ordo Animi: command and orchestration layer.
- PMO Portal: governed engagement and artefact lifecycle workspace.
- EA Artefact Generator: enterprise architecture artefact limb.
- BA Artefact Generator: business architecture and analysis artefact limb.
- SA Artefact Generator: solution architecture artefact limb.
- PM Artefact Generator: programme, project, and delivery-management artefact limb.

## Operating principle

AI drafts. Human reviews. PMO governs. Controlled artefacts are shared only after review.

## Minimum maturity standard

Each system should have:

- Clear product intent.
- Defined user audience.
- Live or deployable route.
- Local run and build instructions.
- Deployment notes.
- VAF / ZenCloud ecosystem fit.
- Recognised professional artefact catalogue.
- Artefact generation or preview mode.
- Artefact output management.
- Copy or download capability.
- Structured export path where relevant.
- Artefact lifecycle states.
- Handoff model to PMO Portal.
- Security and data-handling notes.
- Roadmap.
- Licence posture.

## Shared lifecycle

All generators should align to this lifecycle, even if implementation is phased:

Required -> Draft -> In Review -> Approved -> Shared -> Locked -> Handover

Minimum MVP states:

- Required.
- Draft.
- Reviewed.
- Approved.

## Alignment rule

A limb is aligned when it can answer:

1. What professional work does it own?
2. What artefacts does it generate?
3. What lifecycle states does it support?
4. How does it hand off to PMO Portal?
5. How does Ordo Animi orchestrate or govern it?
6. What data is safe to enter?
7. How is output exported, reviewed, approved, shared, and locked?
