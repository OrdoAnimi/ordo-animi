# VALOUR Value Stream Map

Version: 0.1
Status: Private value stream map
Phase: Architecture and Pilot Build

## Purpose

This document defines the value streams for VALOUR for Architecture Leadership.

The value stream map shows how VALOUR turns an architecture leadership situation into preparation, rehearsal, improved behaviour, review, evidence, and product learning.

## Primary value stream

The primary value stream is:

Identify leadership moment -> Prepare -> Rehearse -> Refine -> Apply -> Review -> Report -> Learn -> Improve product.

## Value Stream 1: Identify leadership moment

### Trigger

A user has an upcoming architecture leadership situation.

### Inputs

- User role
- Work context
- Upcoming meeting or conversation
- Desired outcome
- Leadership challenge

### Activities

- Complete intake
- Identify scenario
- Confirm fit
- Establish confidence baseline

### Outputs

- User intake record
- Selected scenario
- Baseline confidence

### Value created

The user stops treating the situation as vague pressure and turns it into a specific leadership moment.

## Value Stream 2: Prepare

### Trigger

Scenario has been selected.

### Inputs

- Scenario record
- Audience
- Decision required
- Risk
- Desired outcome

### Activities

- Clarify decision
- Identify stakeholders
- Identify likely questions
- Define key message
- Define tone
- Define closing statement

### Outputs

- Preparation brief

### Value created

The user enters rehearsal with a clear frame rather than scattered thoughts.

## Value Stream 3: Rehearse

### Trigger

Preparation brief is ready.

### Inputs

- Preparation brief
- Scenario prompt
- User answer

### Activities

- Simulate questions
- Practise responses
- Test clarity
- Test cadence
- Test empathy
- Test authority
- Test decision framing

### Outputs

- Rehearsal notes
- Feedback
- Improvement areas

### Value created

The user experiences the difficult part before the real meeting.

## Value Stream 4: Refine language

### Trigger

A user answer needs improvement.

### Inputs

- User answer
- Scenario context
- Rehearsal feedback

### Activities

- Shorten message
- Reframe for audience
- Add empathy where needed
- Strengthen decision language
- Reduce technical overload

### Outputs

- Short version
- Executive version
- Empathetic version
- Decision version

### Value created

The user gets language they can actually use.

## Value Stream 5: Apply

### Trigger

The real meeting or conversation occurs.

### Inputs

- Preparation brief
- Rehearsal learning
- Refined language
- User judgement

### Activities

- Attend meeting
- Use prepared frame
- Handle questions
- Clarify decision
- Preserve trust

### Outputs

- Real-world outcome

### Value created

The pilot moves from theory into actual leadership behaviour.

## Value Stream 6: Review

### Trigger

The real situation has completed.

### Inputs

- User recollection
- Outcome
- Stakeholder response
- Preparation materials

### Activities

- Capture what happened
- Identify what worked
- Identify what failed
- Review clarity and cadence
- Review empathy and authority
- Identify next improvement

### Outputs

- After-action review

### Value created

The user turns experience into learning.

## Value Stream 7: Report

### Trigger

After-action review is complete.

### Inputs

- Intake
- Scenario record
- Rehearsal notes
- Review notes
- User feedback

### Activities

- Identify recurring pattern
- Identify strength
- Identify risk
- Recommend next focus
- Capture usefulness score

### Outputs

- Pattern report
- Pilot completion record

### Value created

The user receives a concise development insight and the product captures validation evidence.

## Value Stream 8: Product learning

### Trigger

Pilot record is complete.

### Inputs

- User feedback
- Evidence metrics
- Scenario usage
- Completion data
- Qualitative comments

### Activities

- Review evidence
- Identify repeated pain
- Identify high-value scenarios
- Identify friction
- Update backlog
- Decide next release

### Outputs

- Product improvement backlog
- Release decision
- Architecture input

### Value created

VALOUR improves based on real usage rather than abstract assumptions.

## Value stream metrics

- Intake completion rate
- Scenario fit score
- Preparation usefulness score
- Rehearsal usefulness score
- Confidence before and after
- Real-world application completed
- Review completed
- Repeat intent
- Recommendation signal
- Team or sponsor signal

## Value stream conclusion

The first VALOUR product does not need every feature.

It needs to protect and improve the value stream from leadership moment to measurable learning.
