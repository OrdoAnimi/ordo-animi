# VALOUR Pilot User Journey

Version: 0.1
Status: Private user journey
Release: Pilot Ready

## Purpose

This document defines the user's experience during the first VALOUR pilot.

It separates what the user sees from what the operator manages behind the scenes.

## User journey summary

The user journey has six visible steps:

1. Understand the pilot
2. Provide intake information
3. Choose or confirm a real scenario
4. Prepare and rehearse
5. Use the preparation in the real world
6. Review and receive a pattern snapshot

## Step 1: Understand the pilot

### User question

What is VALOUR and why should I try it?

### User experience

The user receives a short explanation:

VALOUR helps architects and technical leaders prepare for real leadership conversations before they happen.

The user is told that the pilot is practical, scenario-based, and focused on one real upcoming conversation.

### Operator action

Share the pilot pack or explain the pilot verbally.

## Step 2: Provide intake information

### User question

What do you need from me?

### User experience

The user completes a short intake form covering role, context, current challenge, upcoming conversation, confidence level, and desired outcome.

### Operator action

Review intake and identify likely scenario fit.

## Step 3: Choose or confirm scenario

### User question

Which situation are we preparing for?

### User experience

The user confirms one real upcoming scenario.

Examples include review board, executive briefing, discovery workshop, delivery conflict, risk discussion, trade-off explanation, or first meeting with a new team.

### Operator action

Map the user's situation to the scenario library.

## Step 4: Prepare and rehearse

### User question

How do I get ready for the real conversation?

### User experience

The user receives a preparation brief and rehearses the difficult part with AI support.

The rehearsal helps the user clarify the audience, message, risk, likely questions, tone, and decision framing.

### Operator action

Run the preparation and rehearsal prompts. Capture useful outputs.

## Step 5: Use preparation in the real world

### User question

What do I do in the actual meeting?

### User experience

The user enters the real meeting with a clearer frame, stronger language, and better awareness of likely risks.

The first pilot does not provide live meeting support.

### Operator action

Wait for the user to complete the real-world situation.

## Step 6: Review and receive pattern snapshot

### User question

What did I learn and what do I improve next?

### User experience

The user completes an after-action review.

The user receives a short pattern snapshot covering what worked, what failed, what pattern appeared, and what to improve next time.

### Operator action

Run after-action review, capture feedback, and prepare the completion report.

## Emotional experience target

The user should feel more prepared, clearer, calmer, and more deliberate.

The user should not feel judged, diagnosed, overwhelmed, or sold to.

## User promise

Bring one real conversation.

Leave with better preparation, better language, and a clearer improvement pattern.

## Operator promise

Keep the pilot focused.

Do not turn one scenario into a full coaching program.

Do not expose unnecessary product complexity.

Do not over-explain Ordo Animi.
