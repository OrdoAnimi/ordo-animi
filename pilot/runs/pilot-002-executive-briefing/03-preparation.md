# Stage 03 — Preparation Brief

## Pilot ID

PILOT-002

## User ID

SIM-002

## Audience

CFO and CIO. Joint briefing. Time: 15 minutes maximum.

CFO priorities: cost containment, risk quantification, return on investment timeline, optionality.
CIO priorities: delivery capability, speed to benefit, architectural control, vendor dependency management.

## Decision frame

This is an investment decision, not an architecture decision. The architecture recommendation has already been made at the technical governance level. This briefing asks the executive team to confirm the investment direction and approve the delivery mandate.

## Key message

We are recommending the managed migration approach. The migration risk is controllable within the existing delivery model, the investment profile is within the approved range, and the cost of deferring the decision is growing with each quarter. The recommendation is to proceed.

## Likely executive questions

1. What is the cost difference between lift-and-shift and re-platform, and which do you recommend?
2. What happens if the migration runs over time or over budget?
3. Can we do this without disrupting the current quarter's revenue commitments?
4. What is the single biggest risk and how are we managing it?
5. If we approve this today, what is the first commitment and when do we see the first result?

## Closing statement

The recommendation is clear and the team is ready to proceed. The migration is a managed investment with quantified risk, not an open-ended technical project. Approving the mandate today gives the programme team the certainty they need to begin mobilisation this quarter.

## Preparation notes

- Lead with the recommendation. Do not build to it.
- Address the CFO and CIO lenses in the first 90 seconds: cost and risk for the CFO, capability and speed for the CIO.
- Never use re-platform, lift-and-shift, or dependency chain language without immediately translating it.
- The answer to every question should end with either a consequence, a risk mitigation, or a next action.

## Operator note
