# Pilot 001 Pattern Summary

## User story

As a VALOUR pilot user, I want a concise summary so that I understand the starting challenge, improvement, recurring pattern, and next development focus.

## Acceptance criteria

- Starting challenge is summarised.
- Scenario used is recorded.
- Improvement is identified.
- Primary pattern is named.
- Next focus is recommended.

## Success criteria

The summary is successful when the user can identify one practical behaviour to carry into the next leadership situation.

## User ID

SIM-001

## Scenario used

Architecture review board for a cloud integration design.

## Starting challenge

The user understood the design but risked losing the board by over-explaining and not making the decision frame explicit.

## What VALOUR helped prepare

VALOUR helped clarify the audience, decision, key message, likely questions, risks to avoid, opening frame, and closing statement.

## Main improvement observed

The user moved from defending the design to framing the board decision as conditional approval with clear trade-offs and next actions.

## Primary pattern detected

Technical confidence without enough decision framing.

## Primary strength

Strong understanding of technical risk, controls, and delivery constraints.

## Primary risk

The user may answer from implementation detail before establishing the leadership frame.

## Recommended next focus

Start every board-level answer with the decision frame first, then explain the supporting detail only as needed.

## Useful language to retain

The recommendation is conditional approval, not unconditional approval.

The trade-off is speed with controlled governance, not speed without governance.

## Pilot usefulness score, 1 to 10

8

## Recommended next step

Build the first VALOUR Pilot Console around intake, scenario selection, preparation brief, practice question, language refinement, review, and summary generation.
