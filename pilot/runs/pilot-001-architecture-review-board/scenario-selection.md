# Scenario Selection

Pilot: Pilot 001 - Architecture Review Board

Status: Complete

## Selected Scenario

Architecture Review Board

## Why This Scenario Fits

The user is preparing for a high-stakes architecture governance conversation. The meeting is not primarily a technical walkthrough. It is a decision forum where senior stakeholders need to understand the recommendation, the viable alternatives, the trade-off, the risk position, and the conditions for approval.

The Architecture Review Board scenario is the correct VALOUR scenario because it tests the exact leadership behaviour the user needs:

- moving from design detail to decision framing
- presenting trade-offs without over-defending
- holding executive altitude
- responding calmly to security, platform, and delivery challenge
- closing with a clear ask

## Scenario Pressure

The pressure comes from three sources:

1. Delivery wants approval quickly.
2. Security wants assurance that controls and auditability are sufficient.
3. Enterprise architecture wants confidence that the pattern does not create avoidable platform debt.

The architect may try to satisfy all three by adding more technical detail. That would increase the risk of losing clarity.

## Success Definition

The scenario is successful when the architect can:

- state the requested decision in the first minute
- explain the recommendation in plain language
- name the trade-off directly
- identify the residual risk without sounding defensive
- answer challenge in short, structured responses
- close by confirming conditions and next actions

## Failure Pattern To Avoid

The scenario fails if the architect:

- opens with implementation mechanics
- treats every design choice as equally important
- gives long answers to narrow questions
- sounds defensive when challenged by security
- leaves the decision implicit
- ends without a clear board action

## Operator Decision

Proceed with Architecture Review Board as the Pilot 001 scenario.

This scenario gives VALOUR a strong test of prepare, rehearse, perform, review, and improve.
